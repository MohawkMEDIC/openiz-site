Immunization Management System

At the heart of OpenIZ lies the Immunization Management System (IMS). The IMS components represent all services related to persistence, forecasting, reporting, integration, and security. Which users, applications and devices access the IMS are up to you!

User Interfaces &
Apps

OpenIZ provides an Open Architecture. Through a series of RESTful web services application can easily access the functionality provided by the IMS. Additionally, the IMS supports standards-based interfaces including HL7® FHIR™, HL7® Version 2, & HL7® CDA™

Private/Public Cloud Ready

OpenIZ has been designed with cloud computing in mind. Whether you wish to host an entire OpenIZ instance locally, remotely or a combination, the Open Architecture of OpenIZ will support your deployment.

Global Scope

OpenIZ is intended to provide a single consistent Immunization Management System (IMS) which will allow local development teams to customize local instances. Through this "global back-end / local front-end" approach, OpenIZ hopes to build local in-country capacity whilst supporting immunization campaigns.

Forecasting

OpenIZ provides a extensible Immunization Management System (IMS) which allows for a variety of forecasting mechanisms to be leveraged. Whether it is a local Business Rules Engine (BRE), an external expert system, or simple JavaScript code, the OpenIZ IMS will support your vaccination schedule.

Immunization Tracking

The OpenIZ is designed to provide comprehensive tracking of all data related to the administration of vaccinations. The OpenIZ data model is flexible supporting:

  • Administrations & Non-Administrations
  • Observations & Adverse Events
  • Patient Demographics & Links
  • Appointments & Encounters
  • Providers & Facilities

Each data element within the master data model (MDM) can be customized to leverage custom terms. OpenIZ adapts to your environment.

Anlysis & Reporting

The OpenIZ Immunization Management System (IMS) package's Report Integration Service Interface (RISI) allows easy integration of qualified client software into server-side reporting engines. Additionally OpenIZ IMS provides extension points which can be used to send data to your Health Management Information System (HMIS)

Privacy by Design

Because OpenIZ can track information related to a patient's HIV status, or immunosuppression status, it has been built from the ground up with privacy enforcement and security considerations. OpenIZ provides customization points which allow implementers to integrate the policy enforcement engine with external consent management systems.
